Editor Window

Variable Information:

For device state payload formation, certain function arguments are available as listed below.

Utilize these arguments to format the payload as expected by the client application.

"deviceName" : Name of the device for which state change is requested

"deviceUid" : UID of the registered device

"deviceStates" : A string of an array containing metadata of the received device states.

deviceStates Payload Format:

{{stateTranspose|json}}

Each element of the array will be a dictionary containing following keys:

"propertyName": Name of the device property for which desired state is requested

"propertyValue" : Desired value of the device property

"timestamp" : Timestamp value in ISO 8601 format at which the desired state is received by Selene.
